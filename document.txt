SESSION 5 — Career Scraper Fix (2026-07-13)
-------------------------------------------
FIXED: services/career_scraper.py — Full overhaul (v2), scraper was returning 0 jobs
  BUG 1 — _JOB_KEYWORDS regex was too narrow
    Added: recruiter, hr, talent, sales, marketing, support, ux, ui, graphic,
           content, writer, finance, accountant, operations, embedded, network,
           system, infrastructure, product, growth, seo, dba, etl, helpdesk,
           administrator, vacancy, role, hiring, and 15+ more terms
  BUG 2 — _NOISE_PATTERNS used .search() instead of .match()
    Old: re.compile(r"(cookie|privacy|...)", re.I)  ← blocked valid job titles
    New: re.compile(r"^(cookie policy|privacy policy|...)$", re.I)  ← exact match
  BUG 3 — SPA detection threshold too low (300 chars)
    Old: len(body_text) < 300 and root_div  ← most JS career pages skipped
    New: threshold 800 chars; also detects via link count < 5
  BUG 4 — _extract_job_links URL filter too restrictive
    Old: required URL to contain "job"/"career"/etc. even for same-domain links
    New: same-domain links allowed freely; only cross-domain links need job URL
  BUG 5 — No redirect URL tracking
    Old: ATS platform detected from original URL only
    New: final_url after httpx redirect is used for ATS detection and parsing
  BUG 6 — Only 7 ATS parsers
    Added: Workable, Recruitee, Teamtailor, Personio, iCIMS, Breezy HR, JazzHR
    Added: _detect_ats_platform() — maps URL patterns to platform names
    Added: _ATS_URL_PATTERNS — dict of ATS regex patterns
    Added: Try ALL ATS parsers on every page (catches embedded ATS widgets)
  BUG 7 — Playwright scraping too shallow
    Old: 1 scroll + 4s wait
    New: 3 scroll passes, wait for job CSS selectors, 5s final wait, 30s timeout

CHANGED: services/career_scraper.py — file grows from 402 → 636 lines

SESSION 6 — Docker Support (2026-07-15)
----------------------------------------
CREATED: Dockerfile
  - Multi-stage build (builder + runtime) based on python:3.11-slim
  - Stage 1 (builder): installs gcc, libffi-dev, libssl-dev, libxml2-dev,
    libxslt1-dev; pip-installs all requirements.txt into /install prefix
  - Stage 2 (runtime): copies /install, installs Playwright Chromium
    with all system dependencies (libnss3, libatk, libgbm, etc.)
  - ENV: PYTHONUNBUFFERED=1, PYTHONDONTWRITEBYTECODE=1, FLASK_ENV=production
  - VOLUME: /root/.jobtracker  (SQLite DB, sessions, backups, settings)
  - EXPOSE 5000
  - HEALTHCHECK: polls http://localhost:5000/ every 30s
  - CMD: flask --app app:create_app() run --host=0.0.0.0 --port=5000

CREATED: .dockerignore
  - Excludes: __pycache__, .venv, *.db, *.sqlite, .env, .git,
    playwright-browsers, .vscode, .pytest_cache, tests/, *.xlsx

CREATED: docker-compose.yml
  - Single service 'jobtracker', build from local Dockerfile
  - Named volume: jobtracker_data → /root/.jobtracker
  - Env vars SECRET_KEY, FLASK_ENV (overridable via .env file)
  - Healthcheck mirrors Dockerfile; restart: unless-stopped

FEATURE: Infopark Jobs Tab — scrapes https://infopark.in/companies-job

CREATED:
  services/infopark_scraper.py
    - scrape_infopark_jobs(search, max_pages, delay): HTTP scraper using httpx + BeautifulSoup
    - Handles pagination via ?page=N query param
    - Stops when page returns "No Jobs." or empty rows
    - scrape_intern_jobs(): convenience wrapper filtering search="Intern"

  blueprints/infopark.py
    - Blueprint: infopark_bp, url_prefix=/infopark
    - GET  /infopark/           → renders infopark.html
    - GET  /infopark/api/jobs   → returns cached jobs as JSON (supports search, intern, page, per_page)
    - POST /infopark/api/refresh → triggers background scrape thread
    - GET  /infopark/api/status  → returns scrape status (idle/running/done/error)
    - Thread-safe in-memory cache (_cache dict + threading.Lock)

  templates/infopark.html
    - Premium dark glassmorphism UI
    - Auto-scrapes on first visit (zero-config)
    - Job cards with: title, company, posted date, deadline badge, intern badge, apply button

    - Keyword search (debounced 350ms, client-side filter on cached results)
    - Pagination (sliding window, 24 jobs/page)
    - Stats strip: total jobs, internship count, companies count
    - Live status bar with animated dots (idle/running/done/error)
    - Skeleton loading cards while fetching
    - 2-second polling during background scrape
