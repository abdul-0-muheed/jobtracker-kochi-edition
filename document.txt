SESSION 5 — Career Scraper Fix (2026-07-13)
-------------------------------------------
FIXED: services/career_scraper.py — Full overhaul (v2), scraper was returning 0 jobs
  BUG 1 — _JOB_KEYWORDS regex was too narrow
    Added: recruiter, hr, talent, sales, marketing, support, ux, ui, graphic,
           content, writer, finance, accountant, operations, embedded, network,
           system, infrastructure, product, growth, seo, dba, etl, helpdesk,
           administrator, vacancy, role, hiring, and 15+ more terms
  BUG 2 — _NOISE_PATTERNS used .search() instead of .match()
    Old: re.compile(r"(cookie|privacy|...)", re.I)  ← blocked valid job titles
    New: re.compile(r"^(cookie policy|privacy policy|...)$", re.I)  ← exact match
  BUG 3 — SPA detection threshold too low (300 chars)
    Old: len(body_text) < 300 and root_div  ← most JS career pages skipped
    New: threshold 800 chars; also detects via link count < 5
  BUG 4 — _extract_job_links URL filter too restrictive
    Old: required URL to contain "job"/"career"/etc. even for same-domain links
    New: same-domain links allowed freely; only cross-domain links need job URL
  BUG 5 — No redirect URL tracking
    Old: ATS platform detected from original URL only
    New: final_url after httpx redirect is used for ATS detection and parsing
  BUG 6 — Only 7 ATS parsers
    Added: Workable, Recruitee, Teamtailor, Personio, iCIMS, Breezy HR, JazzHR
    Added: _detect_ats_platform() — maps URL patterns to platform names
    Added: _ATS_URL_PATTERNS — dict of ATS regex patterns
    Added: Try ALL ATS parsers on every page (catches embedded ATS widgets)
  BUG 7 — Playwright scraping too shallow
    Old: 1 scroll + 4s wait
    New: 3 scroll passes, wait for job CSS selectors, 5s final wait, 30s timeout

CHANGED: services/career_scraper.py — file grows from 402 → 636 lines
